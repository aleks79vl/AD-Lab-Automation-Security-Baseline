# Security Baseline

## Overview

This document describes the security baseline generated by the project.

The goal is to apply basic Active Directory and Windows hardening settings for lab environments.

## Password Policy

The project generates a domain password policy with:

- Minimum password length: 12
- Password complexity: Enabled
- Password history: 24
- Maximum password age: 90 days
- Minimum password age: 1 day
- Account lockout threshold: 5 failed attempts
- Lockout duration: 30 minutes
- Lockout observation window: 30 minutes

## Workstation Security Baseline

Generated settings include:

- Auto lock screen after 5 minutes of inactivity
- Restrict Control Panel access
- Disable CMD for standard users
- Restrict PowerShell execution for standard users

## Server Security Baseline

Generated settings include:

- Auto lock screen after 5 minutes
- Restrict anonymous enumeration
- Require SMB signing
- Limit blank password usage

## Active Directory Security Logic

The project follows common AD administration principles:

- Users are assigned to groups instead of direct permission assignment.
- GPOs are linked to OUs.
- Workstations and servers have separate baselines.
- Rollback commands are generated separately from deployment.

## Limitations

Standard Active Directory password policy does not support direct banned-word lists such as:

- admin
- administrator
- guest
- user

To enforce banned passwords, an organization would need:

- Microsoft Entra Password Protection
- Custom password filter DLL
- Third-party password policy solution
